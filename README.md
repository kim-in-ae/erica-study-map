✔️프로젝트 필요성

① 공간 정보가 흩어져 있어 탐색에 많은 시간이 소요됨

-도서관, 라칼, 공학관 등 학습공간이 많지만 각 건물별 위치·시설·운영시간 등의 정보가 하나로 모여있지 않음

-학생들은 직접 여러 건물을 돌아다니며 빈 자리를 확인해야 했음

-특히 시험기간에는 이 비효율이 더 크게 체감됨

② 분위기·소음·좌석 구조 등 질적 정보가 부족함

-조용한 공간을 찾고 싶은데 실제로는 팀플 공간인 경우도 많음

-열람실은 정숙 공간인지, 키보드 사용 가능한지 정보를 찾기 어려움

-학생 각자 선호도가 다르지만, 선택을 도와줄 지표가 부재

③ 신입생과 전과생, 교환학생에게 특히 불친절한 구조

-넓고 복잡한 ERICA 캠퍼스 특성상 처음 학기를 시작한 학생들은 공부 장소를 찾는 데 어려움

-‘어디 가서 공부해?’라는 질문이 매 학기 반복됨

-전과생/복수전공생은 다른 건물 공간 정보를 거의 모르는 경우가 많음

④ 팀플·스터디 공간의 실사용 정보 부족

-“화이트보드 있는 곳?”, “대화 가능한 곳?”, “모니터 있는 곳?” 이런 정보가 학생들끼리 구전으로만 전달됨

-공간 예약 필요 여부도 미리 알기 어려움

⑤ 캠퍼스 공간은 많지만 ‘발견되지 않는’ 문제

-뜨끈하냥, 아틀리에, 숨겨진 라운지 등 실제 좋은 공간이 많지만 유명하지 않아 사용되지 않는 공간이 존재

-공간은 남는데 특정 구역만 과밀해지는 비효율 발생

⑥ 교내 공간 운영 리소스 분산 문제

-혼잡 구역으로만 몰리는 현상을 완화할 필요

-학생들이 다양한 스팟을 활용하면 공간 배치·운영의 효율이 증가


📌프로젝트 중요성

① 지도 기반 UI로 ‘한눈에 찾는’ 공간 탐색 경험 제공

건물별·구역별로 분산된 정보를 직관적이고 감성적인 한 페이지 UI에 모아 시각화

② 각 공간별 실사용 정보를 태그로 구조화

예: #정숙 #대화가능 #키보드가능 #따뜻 #카페감성 #24시간 #집중

→ 학생들이 자신의 학습 스타일에 맞는 공간을 빠르게 찾도록 설계

③ 실제 사진 중심의 콘텐츠 제공

텍스트 정보보다 더 직관적이면서 공간 분위기, 좌석 구조, 채광 등을 판단할 수 있음

→ “실제로 가봐야 알 수 있음” 문제 해결

④ 스크롤·애니메이션 기반 UX로 더 쉽게 탐색하도록 개선

fade-in 효과

카테고리 필터링

하위 장소 토글 기능

→ 사용자 경험을 학습 콘텐츠 탐색에 최적화

⑤ ‘탐험 중인 미지의 스팟’ 기능으로 확장성을 확보

새로운 공간을 지속적으로 추가할 수 있는 구조

사용자 참여 기반 데이터로 확장 가능

→ 지속적인 프로젝트 성장성 어필


✅ 기능 설명

캠퍼스 지도 기반으로 캠퍼스 내 스터디 장소 마커 표시

각 장소 정보 표시 : 이름, 위치, 운영시간, 설명

장소 데이터는 JSON 형식으로 분리되어 있어, 유지 보수 및 추가가 용이

🌐 협업 과정 (역할분담)

역할 1) 지도 렌더링, 마커 표시
역할 2) UI, 스타일링, 페이지 구조(HTML,CSS)
역할 3) 장소 데이터 수집 및 JSON 작성, 문서화, 배포 설정
